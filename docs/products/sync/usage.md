# 使用（Sync）

## 创建任务

一般流程：

1) 配置 OpenList（存储/账号/接口等）
2) 创建任务：源目录 → 目标目录
3) 选择是否启用“文件处理”（如 MD5 修改）
4) 保存并启用任务

## 观察进度

- 任务与文件进度会通过 **SSE** 实时推送。
- 你也可以结合容器日志排查问题。

## 日志

```bash
# 容器日志
sudo docker logs -f panbox-sync

# 若使用脚本默认路径，可查看数据目录下日志文件
sudo tail -f /opt/panbox-sync/data/logs/app.log
```

## 失败与重试

流水线常见失败点：

- 下载失败（网络/权限/源端限制）
- 处理失败（文件格式/权限/磁盘空间）
- 上传失败（目标端限制/空间不足）

建议先在小规模任务验证配置，再扩大并发与范围。
