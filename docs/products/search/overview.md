# PanBox Search - 网盘搜索站 概览

PanBox Search 是一个面向网盘资源检索的自托管搜索站点，用于对外提供统一搜索入口，先把搜索和分享流量接住，再把稳定资源持续分发出去。

## 产品定位

Search 在 PanBox 体系中承担前台入口的角色，主要负责：

- 对外提供可访问的网盘搜索页面
- 承接用户的搜索、浏览与访问需求
- 为 Autosave / Sync 沉淀和处理后的资源提供统一展示面

如果你现在最需要的是先搭一个能承接流量、能持续展示资源的站点，通常就从 Search 开始。

## 适用场景

- 想搭一个公开可访问的资源站、搜索站或私域资源入口
- 想把分散的资源入口统一到一个站点中，减少用户流失
- 已经有资源来源，但缺少一个稳定的对外展示与分发页面

## 核心能力

- 提供基于 Web 的资源搜索与聚合检索能力
- 适合作为资源站、分享站或私域资源库的统一入口
- 可承接 Autosave 沉淀后的资源与 Sync 处理后的稳定资源

Search 解决的是“用户怎么先找到你、先看到你的资源”，不是资源沉淀和保活本身。

## 部署形态

PanBox Search 当前是 **PHP / ThinkPHP6** 形态，更接近传统 Web 应用：

- 容器内运行 PHP 应用与相关依赖
- 部署方式更偏网站应用，而不是任务型后端服务
- 如果你熟悉 Web Server + 数据库的部署方式，上手会更自然

## 与其他产品的关系

- `Search`：负责搜索入口与结果展示
- `Autosave`：负责把外部分享链接持续转存成自己的资源库存
- `Sync`：负责跨网盘同步、处理、自动转码与资源保活

三者可以单独使用，但组合后更容易形成“流量入口 -> 资源沉淀 -> 稳定分发”的完整链路。
